# 真实数据采集说明

## 1. 当前正式方案：国家流感中心北方省份周度序列

项目已从“北京市平谷区代理北京市”迁移到更稳定的公开数据口径：

- 流感主序列：中国国家流感中心《中国流感监测周报》
- 研究地区：中国北方省份
- 目标变量：`ili_rate`，北方省份哨点医院报告的 ILI%
- 辅助流感字段：`positive_rate`，北方省份流感病毒检测阳性率。该字段保留原始留痕；若 PDF 表格不可稳定抽取，默认训练可先不使用。
- 时间跨度目标：流感目标序列覆盖 2010-2026 年；多源建模默认从百度指数可用的 2011 年开始

该方案优于山东省单省方案：国家流感中心周报按南方/北方长期发布，样本量更大，口径更稳定，公开页面可追溯；目前未发现山东省公开、连续、逐周、结构化程度相当的数据源。

## 3. 国家流感中心周报采集

运行：

```powershell
python scripts/fetch_flu_cn_weekly.py --start-year 2010 --end-year 2026
```

脚本逻辑：

1. 读取国家流感中心新版周报列表：`https://ivdc.chinacdc.cn/cnic/zyzx/lgzb/`
2. 读取旧版周报列表：`https://ivdc.chinacdc.cn/lgzx/zyzx/lgzb/`
3. 从 HTML 正文抽取 `ili_rate` 与 `positive_rate`
4. 若近年页面只保留摘要，则自动下载并解析 PDF 附件
5. 输出 CSV 与解析质量报告

输出文件：

- `data/raw/flu/cnic_north_weekly_flu.csv`
- `results/reports/cnic_weekly_parse_report.json`

如果想先快速检查 HTML 能解析多少：

```powershell
python scripts/fetch_flu_cn_weekly.py --start-year 2010 --end-year 2026 --no-pdf
```

## 4. 气象数据

当前配置使用 Open-Meteo Archive API 自动拉取北方代表城市日度气象数据并聚合：

- 北京、天津、济南、石家庄、沈阳、哈尔滨、西安、太原

输出路径：

- `data/raw/weather/north_representative_city_weather.csv`

必要字段：

```csv
date,temperature,humidity,wind_speed,pressure
```

## 5. 百度指数数据

百度指数需要登录态 Cookie。推荐整理北方代表城市或省份组合，而不是继续只用北京：

- 北京、天津、济南、石家庄、沈阳、哈尔滨、西安、太原

输出路径：

- `data/raw/search/north_baidu_index.csv`

必要字段：

```csv
date,flu_search_index,cold_search_index,fever_search_index
```

聚合方式建议优先级：

1. 人口权重聚合
2. 简单平均
3. 若部分城市缺失，先记录缺失再按可用城市平均

## 6. 最优执行顺序

1. 跑 `scripts/fetch_flu_cn_weekly.py`，先拿到真实周度目标序列。
2. 检查 `cnic_weekly_parse_report.json`，优先确认 `ili_rate` 缺失数是否为 0；`positive_rate` 可作为质量说明或后续增强字段。
3. 生成 `north_baidu_index.csv`。
4. 运行 `python scripts/train.py`，自动拉取/缓存北方代表城市气象数据。
5. 运行 `python scripts/run_ablation.py --skip-collect`。
6. 重新生成论文初稿，并彻底删除旧平谷代理和 synthetic_extension 的最终结论。
